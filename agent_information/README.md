# À quoi sert ce dossier

Documentation interne du projet : contexte, problèmes déjà rencontrés et leurs causes,
détail des sources de données. Le but est d'éviter de refaire les mêmes erreurs et de
perdre du temps à re-comprendre des choses déjà établies.

Utile à toute personne qui reprend le projet, et en particulier aux agents de
développement IA (Codex, Claude Code, etc.) : ça leur donne directement le contexte
nécessaire sans avoir à re-explorer tout le dépôt à chaque session.

- [project_summary.md](project_summary.md) — vue d'ensemble du projet, structure, pipeline.
- [error_log.md](error_log.md) — erreurs connues et déjà résolues, avec leur cause.
- [data_sources.md](data_sources.md) — détail technique des sources de données utilisées.
